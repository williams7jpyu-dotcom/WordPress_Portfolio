学びの記録Notes — Learning log

プログラミング学習で詰まったこと、解決した方法、設計で考えたこと。手を動かして気づいたことを、自分の言葉で書き留めています。前職の事務仕事と地続きだった、という発見が多めです。

2026.05.18 MVCがなかなか掴めなかった話「Controllerに書くのか、Modelに書くのか」で毎回手が止まっていた。原因はコードじゃなかった。 Java 2026.04.27 JOINでデータが増えてしまった日10件のはずの一覧が30件に。SQLじゃなくて、テーブルの持ち方への理解だった。 Database 2026.03.30 初めてのmergeコンフリクト5人のチーム実習。赤い文字でCONFLICTと出て、最初は事故かと思った話。 Tools 2026.03.08 例外を握りつぶさないとりあえずtry-catchで囲む。それは問題を解決ではなく、隠していただけだった。 Java 2026.02.14 CSSの余白設計は、紙面づくりと同じだったmarginを目分量で足すと散らかる。前職で資料を整えていた感覚がそのまま効いた。 CSS 2026.01.22 チームリーダーとして工程表を引いたコードは一番できるわけじゃない。でも、段取りなら8年やってきた。 Team 2025.12.05 なぜ事務職から、コードを書く側へ8年間、正確に「処理する」仕事をしてきた。これからは、その正確さで「つくる」側に。 Career
