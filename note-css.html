CSS2026.02.14読了 約4分

CSSの余白設計は、
紙面づくりと同じだった

marginを目分量で足していくと、画面はどんどん散らかる。これ、前職で資料を整えていたときと同じだ、と気づいた。

CSSを習い始めて、レイアウトは作れるようになった。でも、なぜか「なんとなく揃っていない」感じが消えない。要素ごとにmargin: 13px、margin: 18px……と、その場の見た目で数字を足していたからだ。

01 — Random numbersバラバラの数字

困ったときに思い出したのが、前職で資料を整えていたときのことだった。読みやすい資料は、見出しと本文の間隔、段落どうしの間隔に決まったリズムがあった。逆に、行間や余白がバラバラな資料は、内容が同じでも素人っぽく見えた。

CSSでも同じだった。余白を「気分」で決めるのをやめて、基準を決めることにした。

02 — A spacing scale余白に基準を持たせる

4の倍数を基本にした「スケール」を作って、余白はそこからしか選ばないルールにした。CSS変数にしておけば、後からまとめて調整もできる。

CSS:root {
  /* 余白は、この中からしか選ばない */
  --space-1: 8px;
  --space-2: 16px;
  --space-3: 24px;
  --space-4: 40px;
}

.card { padding: var(--space-3); }
.card + .card { margin-top: var(--space-2); }

これだけで、画面が一気に整って見えるようになった。バラバラの数字を消して、選択肢を絞る。自由をなくした方が、きれいになる——これも資料づくりで学んでいたことだった。

余白は「足すもの」じゃなくて「選ぶもの」。基準があると、迷いも、ばらつきも減る。

03 — Gap over margin並べる余白はgapで

もう一つ変えたのが、要素を横や縦に並べるとき。以前は一つひとつにmarginを付けていたけれど、flexやgridのgapを使うと、間隔を一括で管理できて、最後の要素の余分な余白に悩むこともなくなった。

CSS.list {
  display: flex;
  flex-direction: column;
  gap: var(--space-2);  /* 間隔はこれ一行 */
}
気づき

デザインのセンスがない、と思っていた。でも実際に効いていたのは、センスより「基準を決めて、それを守る」という、前職でずっとやってきたことだった。
